# GRC Risk Register Project

## Overview

This project is a sample Governance, Risk, and Compliance (GRC) risk register designed to demonstrate how IT and cybersecurity risks can be documented, scored, mapped to controls, assigned owners, and tracked through remediation.

The project is built around a fictional healthcare technology environment and focuses on practical risk management concepts such as identity and access management, vulnerability management, endpoint security, third-party risk, audit readiness, and control documentation.

## Purpose

The purpose of this project is to show how security risks can be translated into clear, business-aligned documentation that supports decision-making, remediation tracking, and audit readiness.

This project demonstrates:

- Risk identification and documentation
- Likelihood and impact scoring
- Risk prioritization
- Control mapping
- Remediation planning
- Evidence tracking
- Ownership assignment
- Audit-ready documentation

## Scenario

A fictional healthcare organization is reviewing its IT and security environment to identify risks that could affect confidentiality, integrity, availability, compliance, and operational continuity.

The organization uses common enterprise systems such as Microsoft 365, Active Directory, endpoint management tools, cloud services, SaaS vendors, and ticketing systems.

## Sample Risk Categories

The risk register includes examples related to:

- Identity and access management
- Delayed user offboarding
- Excessive user permissions
- Missing or inconsistent MFA enforcement
- Vulnerability remediation delays
- Endpoint patch compliance gaps
- Incomplete asset inventory
- Third-party vendor security review gaps
- Security alert documentation gaps
- Lack of formal exception tracking

## Risk Scoring Method

Risks are scored using a simple likelihood and impact model.

| Score | Rating | Description |
|---|---|---|
| 1 | Low | Unlikely or limited impact |
| 2 | Medium | Possible occurrence or moderate impact |
| 3 | High | Likely occurrence or significant impact |

Risk Score = Likelihood x Impact

| Score Range | Risk Level |
|---|---|
| 1-2 | Low |
| 3-4 | Medium |
| 6-9 | High |
